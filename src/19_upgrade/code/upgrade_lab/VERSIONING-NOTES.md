# 第十七章实战 · 共享对象「版本字段」迁移（设计稿）

## 建议字段

在共享对象中增加 `schema_version: u64`，升级时：

1. 新代码只**追加** `public fun migrate_v1_to_v2(...)` 等，由管理员事务显式调用；或
2. 在首次访问时若 `schema_version < N` 则原地升级（注意重入与 gas）。

## 升级时禁止

- 改变已有 struct **字段顺序**或删除字段（破坏 BCS 布局）。
- 在共享对象上静默改类型（应新类型 + 迁移路径）。

## 与 `SCHEMA_VERSION` 常量

本包 `version.move` 中的常量用于文档化；链上对象内的版本号需单独存储在对象字段中。
